组件 函数式的编程

react 核心：组件 精髓：函数式编程
react 是谷歌开发的，之前不是，是facebook 在遇见的问题，：虚拟dom操作，数据流向的控制 这两大优点，facebook起初调研过市场上已存的mvc框架，发现都不太满意，于是就推陈出新，开发了react框架，并在2013年五月份开源

React 把用户界面抽象成一个个组件，如按钮组件 Button、对话框组件 Dialog、日期组件 Calendar。开发者通过组合这些组件，最终得到功能丰富、可交互的页面。通过引入 JSX 语法，复用组件变得非常容易，同时也能保证组件结构清晰。有了组件这层抽象，React 把代码和真实渲染目标隔离开来，除了可以在浏览器端渲染到 DOM 来开发网页外，还能用于开发原生移动应用

优势：

1.声明式设计 −React采用声明范式，可以轻松描述应用（自动dom操作）。
2.高效 −React通过对DOM的模拟(虚拟dom)，最大限度地减少与DOM的交互。
3.灵活 −React可以与已知的库或框架很好地配合。可以和第三方库混合使用，vue不可以
4.JSX − JSX 是 JavaScript 语法的扩展。时核心组件的语法
5.组件 − 通过 React 构建组件，使得代码更加容易得到复用，能够很好的应用在大项目的开发中。
6.单向响应的数据流 − React 实现了单向响应的数据流，从而减少了重复代码，这也是它为什么比传统数据绑定更简单。

React的核心是组件，组件的设计目的是提高代码复用率、降低测试难度和代码复杂度。
提高代码复用率：组件将数据和逻辑封装，类似面向对象中的类。
降低测试难度：组件高内聚（当前专注于做自己唯一的功能）低耦合（组件和组件之前的关联性很低），很容易对单个组件进行测试。
降低代码复杂度：直观的语法可以极大提高可读性。（组件是开发定义是直观的语法）

#使用典型代表公司：
国外使用React的公司：Facebook，Flipboard，Airbnb，BBC，GitHub，Instagram，Reddit，Uber，WhatsApp，Yahoo
国内使用React的公司：支付宝，淘宝，大搜车，Teambition，豆瓣，豌豆荚

React 并不是完整的 MVC/MVVM 框架，它专注于提供清晰、简洁的 View（视图）层解决方案。而又与模板引擎不同，React 不仅专注于解决 View 层的问题，又是一个包括 View 和 Controller 的库。对于复杂的应用，可以根据应用场景自行选择业务层框架，并根据需要搭配 Flux、Redux、GraphQL/Relay 来使用。
React 不像其他框架那样提供了许多复杂的概念与烦琐的 API，它以 Minimal API Interface 为目标，只提供组件化相关的非常少量的 API。同时为了保持灵活性，它没有自创一套规则，而是尽可能地让用户使用原生 JavaScript 进行开发。只要熟悉原生 JavaScript（面向对象，原型继承） 并了解重要概念后，就可以很容易上手 React 应用开发。

react 的编程模式，是函数式编程（Functional Programming
）
工业界的编程：方式一直以命令式编程为主。命令式编程解决的是做什么的问题。
函数式编程：对应的是声明式编程，它是人类模仿自己逻辑思考方式发明出来的。
函数式编程好处：
代码简洁，开发快速
接近自然语言，易于理解
更方便的代码管理
易于”并发编程“
代码的热升级

xtml 和HTML 的区别：

1.区分大小写 2.合理嵌套 3.对于标签，必须是双标签
JSX 将 HTML 语法直接加入到 JavaScript 代码中，再通过翻译器转换到纯 JavaScript 后由浏览器执行。在实际开发中，JSX 在产品打包阶段都已经编译成纯 JavaScript，不会带来任何副作用，反而会让代码更加直观并易于维护。
编译过程由Babel 的 JSX 编译器实现。

状态

React 把组件看成是一个状态机（State Machines）。通过与用户的交互，实现不同状态，然后渲染 UI，让用户界面和数据保持一致。状态变更是为了实现不同页面的渲染，一个状态树对应一个新的页面，在react 所有试图的更新，页面的变更，只需要更改状态，根据状态重新渲染，不用操作dom元素

React 里，只需更新组件的 state，然后根据新的 state 重新渲染用户界面（不要操作 DOM）。
