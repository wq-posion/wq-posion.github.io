使用react-redux组件
安装依赖：npm i react-redux –save-dev
https://github.com/reactjs/react-redux
也叫UI组件（presentational component），纯组件
特点：
只负责 UI 的呈现，不带有任何业务逻辑
没有状态（即不使用this.state这个变量）
所有数据都由参数（this.props）提供
不使用任何 Redux 的 API

UI 组件的例子：

const Title = value => <h1>{value}</h1>;

容器组件
负责管理数据和逻辑
特点：
负责管理数据和业务逻辑，不负责 UI 的呈现
带有内部状态
使用 Redux 的 API（80%）

React-Redux 提供connect方法，用于从 UI 组件生成容器组件。connect的意思，就是将这两种组件连起来。

示例代码：

import { connect } from 'react-redux'
const VisibleTodoList = connect(
  mapStateToProps, 
  mapDispatchToProps
)(TodoList);

TodoList是 UI 组件，VisibleTodoList就是由 React-Redux 通过connect方法自动生成的容器组件。

mapStateToProps

作用：
建立一个从（外部的）state对象到（UI 组件的）props对象的映射关系。
mapStateToProps会订阅 Store，每当state更新的时候，就会自动执行，重新计算 UI 组件的参数，从而触发 UI 组件的重新渲染。
例子：

const mapStateToProps = (state, ownProps) => {
  return {
    todos: getdataFromState(state.todos, state.filter)
  }
}

参数说明：
state：状态
ownProps：容器组件的props对象

mapDispatchToProps

作用：
用来建立 UI 组件的参数到store dispatch方法的映射。也就是说，它定义了哪些用户的操作应该当作 Action，传给 Store。它可以是一个函数，也可以是一个对象。
mapDispatchToProps方法的形式：

const mapDispatchToProps = (
  dispatch,
  ownProps
) => {
  return {
    onClick: () => {
      dispatch({
        type: 'SET_VISIBILITY_FILTER',
        filter: ownProps.filter
      });
    }
  };
}
